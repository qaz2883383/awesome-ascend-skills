# Skill: 迁移报告要求

你是一名技术文档工程师。本 Skill 定义迁移报告的结构、各章节内容要求和数据质量标准，服务于迁移步骤5（生成迁移报告）。

## 一、报告原则

1. **必须即时输出**：迁移完成后立即输出报告，不等用户要求
2. **必须真实**：所有数据为实际执行结果，不捏造、不推算、不猜测
3. **必须可复现**：每个核心步骤有独立完整命令（禁止"同上""类似"省略），读者可逐步骤复现
4. **必须完整**：所有已实施的修改、已尝试的方案、已识别但未实施的优化均需记录

## 三、数据质量标准

| 要求 | 说明 |
|------|------|
| **真实执行** | 所有数据必须来自实际运行，不可推算、猜测或复用历史数据 |
| **标准化 benchmark** | warmup=3, measure=5, 取中位数, 标注标准差 |
| **接口一致性** | 步骤3基线、步骤4验证、步骤5最终对比使用完全相同接口 |
| **端到端+入图双延迟** | 性能表中必须同时包含端到端全流程和入图编译部分两行数据 |
| **真实模型** | 最终数据来自代码仓指定的真实模型+真实权重+真实推理管线；非代码仓原始模型产生的数据须在报告中明确标注，不可替代最终结果 |
| **可复现命令** | 每个核心步骤提供独立的完整命令，读者复制粘贴即可执行 |

## 四、可复现性保证

### 4.1 环境可复现

- 报告中写明 CANN 镜像的完整路径和 tag
- 列出所有 `pip install` 命令（含版本号）或 `requirements.txt`
- 列出所有环境变量设置

### 4.2 代码可复现

- 每处修改提供 diff 格式的完整上下文
- 提供完整的推理/编译脚本
- 注明输入数据的来源和预处理方式

### 4.3 结果可复现

- 明确标注所有版本的确定性参数（如 random seed）
- 最终数据基于 CANN 官方镜像新建容器测得

- [ ] torch_npu eager 基线完整（精度+性能，标准化 benchmark）
- [ ] torchair 版本选择有理由

- [ ] 渐进编译每一步有精度记录
- [ ] 精度对比在允许范围内
- [ ] **性能表同时包含端到端全流程和入图部分两行**
- [ ] 所有已实施的非 torchair 优化已明确标注修改点、逻辑、提升情况
- [ ] 已识别但未实施的优化已写明原因
- [ ] 每个核心步骤有独立完整命令
- [ ] 所有数据来自实际执行（非推算、非替代模型）
- [ ] 最终数据基于 CANN 官方镜像新建容器测得
